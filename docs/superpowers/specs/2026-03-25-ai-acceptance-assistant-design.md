# AI 验收助手（AcceptBot）— 设计文档

> 上传需求文档，AI 秒生验收清单，在线逐项验收，一键导出报告

## 背景

用户具备 10 年软件测试 + 4 年项目管理 + 人事采购经验，非开发背景，每周可投入不到 5 小时。验收测试流程中"写验收用例、逐项比对、记录问题、回归验证"每个环节都有痛点。本产品聚焦国内市场，用 AI + 用户的测试经验解决验收效率问题。

## 产品定义

**定位**：面向测试团队 Leader、项目经理、产品经理和甲方验收人员的 AI 验收工具。

**目标用户**：
- 测试团队负责人 / QA Lead
- 项目经理 / PMO
- 产品经理（UAT 验收）
- 甲方验收外包交付物的负责人

**核心用户流程**：
1. 用户上传 PRD / 需求文档（Word/PDF/飞书文档链接）
2. AI 解析文档，生成结构化验收清单
3. 用户在线逐项勾选（通过/不通过/跳过），可添加备注
4. 一键导出验收报告（PDF/Excel）

**验收清单结构**：

| 层级 | 内容 | 示例 |
|------|------|------|
| 功能模块 | 按 PRD 章节自动分组 | 「用户注册」「订单管理」 |
| 检查项 | 每个功能的具体验收点 | 「手机号注册流程正常」 |
| 边界条件 | AI 自动补充的边缘场景 | 「手机号重复注册提示」 |
| 优先级 | P0/P1/P2 | 核心流程 vs 细节体验 |

**MVP 范围**：
- 支持上传 Word/PDF 文档
- AI 生成验收清单（可编辑）
- 在线勾选验收
- 导出 PDF 验收报告

**MVP 不做**：
- 不做截图标注
- 不做 Bug 流转
- 不做回归跟踪
- 不做团队协作
- 不做飞书/钉钉集成

## 技术方案

以「AI 辅助搭建、维护简单」为原则，适配非开发背景。

**前端**：
- 用 Bolt.new 或 Cursor 生成
- React + TailwindCSS
- 单页应用

**后端**：
- Supabase（数据库 + 认证 + 文件存储，一站式）
- 免费额度足够 MVP 阶段

**AI 能力**：
- DeepSeek API（便宜、中文能力强）
- Prompt 是核心资产 — 测试经验转化为 prompt 模板
- 按量付费，成本可控

**文档解析**：
- PDF/Word 上传后提取文本
- 用现成库处理（pdf-parse、mammoth 等）

**部署**：
- Vercel（前端免费托管）
- Supabase 云端（后端免费额度）
- 域名：约 ¥60/年

**月度成本估算**：

| 项目 | 费用 |
|------|------|
| Vercel | 免费 |
| Supabase | 免费（MVP 阶段） |
| DeepSeek API | ¥10-50（按量付费） |
| 域名 | ¥5 |
| **合计** | **¥15-55/月** |

## 变现模式

**定价策略（免费增值）**：

| 层级 | 价格 | 额度 |
|------|------|------|
| 免费版 | ¥0 | 每月 3 份文档解析 |
| 基础版 | ¥29/月 | 每月 30 份文档 |
| 专业版 | ¥79/月 | 无限文档 + 历史记录 + 团队共享 |

**定价逻辑**：
- 免费版零门槛试用，体验 AI 生成清单的价值
- 3 份/月刚好不够用（一个项目多个模块），自然转化付费
- ¥29 是国内工具类产品心理舒适区
- 专业版留给后续，MVP 阶段只做免费版

**支付**：微信支付/支付宝

**MVP 阶段策略**：先不收费，攒 100 活跃用户后再开付费。

## 获客与推广

**冷启动（第一批 100 用户）**：
1. 朋友圈/微信群 — 行业人脉直接推试用
2. 测试社区 — TesterHome、掘金、CSDN 发经验帖带产品
3. 知乎 — 回答「如何做好验收测试」「PRD 验收清单怎么写」等问题

**持续获客**：
1. 小红书 — 「验收测试效率提升 10 倍」「AI 自动生成验收清单」类内容
2. SEO — 「验收测试清单模板」「需求验收工具」等关键词
3. B 站 — 3-5 分钟演示视频，展示完整流程

**内容策略**：
- 不卖工具，卖方法论：「10 年测试老兵的验收方法论，顺便做了个工具」
- 每篇内容输出一个验收经验点，文末带工具链接
- 重点打「AI + 测试经验」的差异化

**第一个月目标**：
- 产品上线
- 微信人脉推 50 人试用
- 发 3 篇内容（知乎/TesterHome）
- 目标：100 注册用户，30 活跃用户

## 时间线与里程碑

| 阶段 | 时间 | 目标 |
|------|------|------|
| 准备 | 第 0 周 | 注册 Supabase、Vercel、DeepSeek，买域名 |
| 原型 | 第 1-3 周 | 用 Bolt/Cursor 搭建 MVP：上传文档 → AI 生成清单 |
| 核心功能 | 第 4-6 周 | 在线勾选验收 + 导出 PDF 报告 |
| 打磨 | 第 7-8 周 | Prompt 调优（用真实 PRD 测试）、UI 细节优化 |
| 上线 | 第 9 周 | 正式上线，微信人脉推第一批用户 |
| 推广 | 第 10-12 周 | 内容发布（知乎/TesterHome/小红书），收集反馈迭代 |
| 评估 | 第 12 周 | 看数据决定下一步 |

**每周 5 小时分配**：
- 开发周：5 小时全部用于搭建
- 推广周：2 小时内容创作 + 3 小时用户沟通和产品迭代

## 成功标准

- 12 周内上线可用产品
- 100 注册用户，30 活跃用户
- 至少 5 个用户主动反馈「有用」
- 12 周后零活跃 → 复盘调整，不恋战

## 与 QA Templates 的关系

- 两个项目可并行，建议交替推进而非同时开发
- QA Templates 的用户和 AI 验收助手的用户高度重合
- 模板商店引流 → 验收工具转化，形成产品矩阵

## 长期路径

MVP 验证后的演进方向：
- 加入截图标注和 Bug 记录 → 完整验收流程平台
- 飞书/钉钉集成 → 嵌入企业协作场景
- 团队协作 + 历史数据 → SaaS 订阅模式
- AI 能力升级：自动对比 PRD 与实际 UI 截图 → 视觉验收
- 终极目标：AI 驱动的软件质量保障平台
