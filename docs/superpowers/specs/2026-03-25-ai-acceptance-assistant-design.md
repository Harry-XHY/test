# AI 验收助手（AcceptBot）— 设计文档

> 上传需求文档，AI 秒生验收清单，在线逐项验收，一键导出报告

## 背景

用户具备 10 年软件测试 + 4 年项目管理 + 人事采购经验，非开发背景，每周可投入不到 5 小时。验收测试流程中"写验收用例、逐项比对、记录问题、回归验证"每个环节都有痛点。本产品聚焦国内市场，用 AI + 用户的测试经验解决验收效率问题。

## 产品定义

**定位**：面向测试团队 Leader、项目经理、产品经理和甲方验收人员的 AI 验收工具。

**目标用户**：
- 测试团队负责人 / QA Lead
- 项目经理 / PMO
- 产品经理（UAT 验收）
- 甲方验收外包交付物的负责人

**核心用户流程**：
1. 用户上传 PRD / 需求文档（MVP 仅支持 PDF，Word 后续迭代）
2. AI 解析文档，生成结构化验收清单
3. 用户在线逐项勾选（通过/不通过/跳过），可添加备注
4. 一键导出验收报告（PDF/Excel）

**验收清单结构**：

| 层级 | 内容 | 示例 |
|------|------|------|
| 功能模块 | 按 PRD 章节自动分组 | 「用户注册」「订单管理」 |
| 检查项 | 每个功能的具体验收点 | 「手机号注册流程正常」 |
| 边界条件 | AI 自动补充的边缘场景 | 「手机号重复注册提示」 |
| 优先级 | P0/P1/P2 | 核心流程 vs 细节体验 |

**MVP 范围**：
- 支持上传 PDF 文档（限制 50 页以内）
- AI 生成验收清单（可编辑：增删检查项、调整优先级）
- 在线勾选验收（通过/不通过/跳过 + 文字备注）
- 导出 PDF 验收报告（含：汇总统计、按模块通过率、逐项明细、备注）

**MVP 不做**：
- 不做 Word/飞书文档解析（迭代 2 加入）
- 不做截图标注
- 不做 Bug 流转
- 不做回归跟踪
- 不做团队协作
- 不做飞书/钉钉集成

## 技术方案

以「AI 辅助搭建、维护简单」为原则，适配非开发背景。

**前端**：
- 用 Bolt.new 或 Cursor 生成
- React + TailwindCSS
- 单页应用

**后端**：
- Supabase（数据库 + 认证 + 文件存储，一站式）
- 认证方式：手机号 + 短信验证码（国内用户习惯），Supabase Auth 支持
- 免费额度足够 MVP 阶段

**AI 能力**：
- DeepSeek API（主力，便宜、中文能力强）
- 备选：通义千问 / GLM-4（防止单一依赖）
- Prompt 是核心资产 — 测试经验转化为 prompt 模板
- 文档超长时采用分段处理：按章节切分后逐段生成清单，再合并
- 按量付费，成本可控

**Prompt 核心策略**：
- 在写任何代码之前，先用 DeepSeek 网页版 + 真实 PRD 手动测试 prompt 效果
- 定义标准化输出格式（JSON），确保前端可解析
- 内置验收经验规则：常见边界条件、安全检查、性能检查等
- 设置质量兜底：AI 输出不合格时提示用户手动补充

**文档解析**：
- MVP 仅处理 PDF，用 pdf-parse 提取文本
- 限制文档大小（50 页 / 5MB 以内）
- 解析失败时提示用户直接粘贴文本

**数据架构**（简化版）：
```
浏览器 → Vercel (React SPA) → Supabase (Auth + DB + Storage) → DeepSeek API
```

**部署**：
- Vercel（前端免费托管）
- Supabase 云端（后端免费额度）
- 域名：约 ¥60/年

**合规要求**：
- ICP 备案（第 0 周启动申请，预计 2-4 周完成）
- 用户文档存储在 Supabase，设置 30 天自动清理策略
- 隐私协议页面：说明数据用途和存储期限

**月度成本估算**：

| 项目 | 费用 |
|------|------|
| Vercel | 免费 |
| Supabase | 免费（MVP 阶段） |
| DeepSeek API | ¥10-50（按量付费） |
| 短信验证码 | ¥10-30（按量） |
| 域名 | ¥5 |
| **合计** | **¥25-85/月** |

## 竞品分析

| 产品 | 定位 | 价格 | 我们的差异 |
|------|------|------|-----------|
| TAPD（腾讯）| 项目管理平台，验收是子功能 | 免费/企业版收费 | 大而全，验收功能弱，不够聚焦 |
| TestLink | 开源测试管理 | 免费 | 需要部署运维，学习成本高 |
| MeterSphere | 开源测试平台 | 免费/企业版 | 面向技术人员，非技术人员用不了 |
| 禅道 | 项目管理 + 测试 | 免费/收费 | 重，功能复杂，验收场景覆盖弱 |

**我们的差异化**：
- 聚焦「验收」这一个场景，不做项目管理
- AI 自动生成清单，零人工编写成本
- 非技术人员也能用（产品经理、甲方）
- 轻量级，上传即用，不需要部署

## 获客与推广

**冷启动（第一批 100 用户）**：
1. 朋友圈/微信群 — 行业人脉直接推试用
2. 测试社区 — TesterHome、掘金、CSDN 发经验帖带产品
3. 知乎 — 回答「如何做好验收测试」「PRD 验收清单怎么写」等问题

**持续获客**：
1. 小红书 — 「验收测试效率提升 10 倍」「AI 自动生成验收清单」类内容
2. SEO — 「验收测试清单模板」「需求验收工具」等关键词
3. B 站 — 3-5 分钟演示视频，展示完整流程

**内容策略**：
- 不卖工具，卖方法论：「10 年测试老兵的验收方法论，顺便做了个工具」
- 每篇内容输出一个验收经验点，文末带工具链接
- 重点打「AI + 测试经验」的差异化

**第一个月目标**：
- 产品上线
- 微信人脉推 50 人试用
- 发 3 篇内容（知乎/TesterHome）
- 目标：100 注册用户，30 活跃用户

## 时间线与里程碑

| 阶段 | 时间 | 目标 |
|------|------|------|
| 准备 | 第 0 周 | 注册账号、买域名、**启动 ICP 备案**（2-4 周）|
| Prompt 验证 | 第 1-2 周 | 用 DeepSeek 网页版 + 真实 PRD 手动测试 prompt，确认 AI 输出质量可用 |
| 原型 | 第 3-6 周 | 用 Bolt/Cursor 搭建 MVP：上传 PDF → AI 生成清单 |
| 核心功能 | 第 7-10 周 | 在线勾选验收 + 导出 PDF 报告 + 手机号登录 |
| 打磨 | 第 11-12 周 | Prompt 持续调优、UI 细节优化、修 bug |
| 上线 | 第 13 周 | ICP 备案完成后正式上线，微信人脉推第一批用户 |
| 推广 | 第 14-16 周 | 内容发布（知乎/TesterHome/小红书），收集反馈迭代 |
| 评估 | 第 16 周 | 看数据决定下一步 |

**每周 5 小时分配**：
- 开发周：5 小时全部用于搭建
- 推广周：2 小时内容创作 + 3 小时用户沟通和产品迭代

## 成功标准

- 16 周内上线可用产品
- 100 注册用户，30 活跃用户（活跃 = 当月完成至少 1 次完整验收流程）
- 至少 5 个用户主动反馈「有用」
- 反馈收集方式：产品内嵌反馈入口 + 建立微信用户群
- 16 周后零活跃 → 复盘调整，不恋战

## 与 QA Templates 的关系

- 两个项目可并行，建议交替推进而非同时开发
- QA Templates 的用户和 AI 验收助手的用户高度重合
- 模板商店引流 → 验收工具转化，形成产品矩阵

## 长期路径

MVP 验证后的演进方向：
- 加入截图标注和 Bug 记录 → 完整验收流程平台
- 飞书/钉钉集成 → 嵌入企业协作场景
- 团队协作 + 历史数据 → SaaS 订阅模式
- AI 能力升级：自动对比 PRD 与实际 UI 截图 → 视觉验收
- 终极目标：AI 驱动的软件质量保障平台
